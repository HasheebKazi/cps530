Password Protection

To password protect the site I made index.html a login page with a simple form. The form will ask for a username and password and then checks it in the browser once the submit button is clicked. The browser will run a script attached to the page which compares the username and password to preset values. If the information provided matches, you get redirected to the lab01.html page. If there is no match you are prompted to try again.

This method is very insecure as a user can simply make a get request for the required page without bothering with the login form. They can also check the client side script to find the correct information.

What new Tag did I try? What was the effect?

I tried the figure tag which is part of the new HTML 5 semantic tags. Using this tag results in a picture properly grouped with a caption. You don't need to use css for the caption to line up under the picture.

Does your page look different in different browsers and platforms(IE/Edge/Firefox/Chrome/Safari - desktop/tablet/phone)? If yes, what are the main differences?

The page looks different when viewed on Firefox and mobile. The main differences are the styling and position of content. On Firefox the login and index pages look very different, the font, fontsize, spacing for elements and button styling is noticibly diferenet. Additionally the default css on Firefox is much nicer than Chrome. For example the button is larger, has extra spacing and a box shawdow to indicate it is a button.

On mobile the page is highly distorted. All the elements are too small to be viewed properly and all elements are spaced vertically which requires additional scrolling.
